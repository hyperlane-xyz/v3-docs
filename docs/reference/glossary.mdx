# Glossary



## Agent

Hyperlane[agents](../protocol/agents/relayer.mdx)是读取和写入Hyperlane智能合约状态的链外参与者。

代理的例子包括[validators](../protocol/agents/validators.mdx) 和[relayer](../protocol/agents/relayer.mdx)。





## Aggregation ISM

[Aggregation ISM](../reference/ISM/aggregation-ISM-interface.mdx)是一种链间安全模块，它通过要求' n '个ISMs中的' m '来验证特定的链间消息，从而聚合来自多个 [ISMs](../reference/ISM/specify-your-ISM.mdx)的安全性。





## Checkpoint

检查点是一个(默克尔根，索引)元组，对应于[mailbox](glossary.mdx#mailbox)增量默克尔树在特定时间点的状态。

[Validators](glossary.mdx#validator) 的检查点签名在[Multisig ISMs](glossary.mdx#multisig-ism)中使用。





## Collateral chain

对于[Warp Route](glossary.mdx#warp-route)， [collateral token](glossary.mdx#collateral-token)存放到Warp Route的链。

阅读更多关于如何 [deploy a Warp Route](../guides/deploy-warp-route.mdx)。





## Collateral token

对于[Warp Route](glossary.mdx#warp-route)，存放于[collateral chain](glossary.mdx#collateral-chain)上的令牌在远程 [synthetic chain](glossary.mdx#synthetic-chain)上创建包装令牌。

阅读更多关于如何[deploy a Warp Route](../guides/deploy-warp-route.mdx)。





## Default ISM

如果消息接收者没有指定自己的ISM，则将使用[interchain security module](glossary.mdx#interchain-security-module) 来验证入站消息。





## Domain

Hyperlane协议用于确定消息来源和目的地的特定链的唯一标识符。

可能与EVM链ID相同，但并不总是如此。请参阅 [domains](domains/) 获取已知Hyperlane域id的列表。





## Interchain accounts (ICA)

一个[middleware](glossary.mdx#middleware)允许用户进行链间智能合约调用的智能合约。

例如，dao可以使用链间帐户来拥有远程链上的合约。

有关更多信息，请参阅跨链帐户。





## Interchain gas paymaster

由[relayer](glossary.mdx#relayer) 部署的智能合约，它在源链上接受支付，以便在目标链上交付消息。

参见 [interchain gas payments](../protocol/interchain-gas-payment.mdx)了解更多信息





## Interchain queries (IQS)

一个[middleware](glossary.mdx#middleware)允许用户进行链间视图调用的智能合约。

例如，智能合约可以使用链间查询来从远程链上查找oracle汇率或代币余额。

参见Interchain Queries (IQS)了解更多信息。





## Interchain security module

[Interchain Security Module](./ISM/specify-your-ISM.mdx) (ISMs)是为Hyperlane的链间[mailbox](./messaging/messaging-interface.mdx)提供安全保障的智能合约。

ISMs负责验证在目的链上传递的链间信息是否**实际发送**在原始链上。





## Mailbox

可以说是Hyperlane最重要的智能合约，[mailbox](./messaging/messaging-interface.mdx) 公开了一个接口，开发人员可以使用它来 [send](./messaging/send.mdx)和 [receive](./messaging/receive.mdx) 链间消息。





## Middleware

发送和接收消息的智能合约，并公开面向开发人员的API。开发人员应该使用这个API，而不是直接与[mailbox](glossary.mdx#mailbox)交互。

示例中间件包括 [interchain-accounts-ica](glossary.mdx#interchain-accounts-ica) 和[interchain-queries-iqs](glossary.mdx#interchain-queries-iqs)





## Multisig ISM

[multisig ISM](./ISM/multisig-ISM-interface.mdx)多重签名 ISM 是一个链间安全模块[interchain security module](glossary.mdx#interchain-security-module) ，它使用来自 n 个验证器[validator](glossary.mdx#validator)中的 m 个验证器的签名来验证特定的链间消息。





## Relayer

[relayer](../operate/relayer/run-relayer.mdx)是一个Hyperlane[agents](glossary.mdx#agent)，负责将消息从源链传递到目标链。

中继器是不可信的，任何人都可以操作中继器。





## Routing ISM

路由ISM是一种[interchain security module](glossary.mdx#interchain-security-module)，根据所传递的消息交由不同的ISM管理。

例如，Routing ISM可以根据发送消息的源链使用不同的ISM。





## Synthetic chain

对于[Warp Route](glossary.mdx#warp-route)，在其上生成包装令牌的链。这些包装的令牌由 [collateral tokens](glossary.mdx#collateral-token)支持，这些令牌已被锁定在 [collateral chain](glossary.mdx#collateral-chain)上。

阅读更多关于如何[deploy a Warp Route](../guides/deploy-warp-route.mdx)。





## Validator

[Validators](../operate/validators/run-validators.mdx) 是一个Hyperlane [agents](glossary.mdx#agent) ，负责验证从源链发送的消息。

[Multisig ISMs](glossary.mdx#multisig-ism) 使用验证器签名为入站消息提供安全性。





## Warp Route

[Warp Routes](applications/warp-routes.mdx#overview)是Hyperlane对令牌桥接概念的采用，允许您通过Hyperlane无许可地将任何类似erc20的资产转移到任何链上。

阅读更多关于如何[deploy a Warp Route](../guides/deploy-warp-route.mdx)。
